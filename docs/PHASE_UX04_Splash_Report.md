# PHASE UX-04 — Professional Legal Splash Screen: Implementation Report

## 1. ملخص الـ Audit

قبل أي تعديل تمت قراءة: `index.html`، `css/components.css`، `css/layout.css`،
`css/base.css`، `css/responsive.css` (لا يوجد `css/animations.css`)،
`js/modules/settings.js`، `js/modules/firstrun.js`.

النتائج:

- شاشة البداية موجودة أصلاً من **PHASE UX-01** (`#splashScreen` في `index.html`،
  ظاهرة افتراضيًا في الـ markup قبل تحميل أي JS حتى لا تظهر صفحة فارغة).
- إخفاؤها يتم حصريًا داخل `js/modules/firstrun.js` عبر الدالة
  `hideSplashAndCheckFirstRun()` التي تضيف الكلاس `splash-hide` على العنصر.
- المنطق الحالي **لا ينتظر Google Sync إطلاقًا**: يوجد `setTimeout` بحد أقصى
  **1500ms** كسقف أمان صلب، بالإضافة إلى `MIN_VISIBLE_MS = 450` تُحسب من لحظة
  تحميل الملف (`window.__splashStart`) لضمان ظهور بصري كافٍ حتى لو كانت
  البيانات المحلية جاهزة فورًا. الاعتماد الوحيد هو جاهزية DOM/البيانات
  المحلية، وليس أي `fetch`/`Promise` جديد.
- لا يوجد `css/animations.css` منفصل — الحركات موجودة داخل `css/base.css`
  و`css/components.css`.
- لا يوجد أي معالجة لـ `prefers-reduced-motion` في أي مكان بالمشروع.
- الشعار القديم كان حرف Unicode `⚖` داخل دائرة تدرّج ذهبي (`.splash-badge`) —
  ليس تصميم SVG/CSS مخصص.
- First Run Wizard يظهر **بعد** اختفاء الـ Splash فقط عبر
  `checkFirstRunWizard()` (ترتيب Splash → اختفاء → Wizard محفوظ كما هو).

**الخلاصة:** لم يكن هناك أي حاجة لتعديل منطق التوقيت أو مصدر البيانات — التعديل
اقتصر على الشكل البصري فقط.

## 3. وصف الـ Animation ومدتها

تسلسل الظهور عند فتح البرنامج (كل مرحلة تبدأ بعد التي قبلها بفارق بسيط):

1. **الشعار**: Fade In + Scale من `0.82` إلى `1` — مدة **500ms**.
2. **الحلقة الذهبية حول الشعار**: Fade بسيط، تبدأ بعد 80ms.
3. **اسم البرنامج (الحسام)**: Fade + TranslateY(6px→0) — تبدأ بعد 180ms، مدة 400ms.
4. **الوصف**: نفس الحركة — تبدأ بعد 300ms.
5. **سطر الإصدار**: نفس الحركة — تبدأ بعد 400ms.
6. **معلومات واتساب**: نفس الحركة — تبدأ بعد 480ms.
7. **الـ Footer السفلي**: Fade فقط — يبدأ بعد 550ms.
8. **الاختفاء (Fade Out)**: يتحكم به `firstrun.js` كما هو (`transition: opacity .45s ease`)
   عند إضافة `splash-hide`.

الحركات مقصورة على `opacity` / `transform: scale()` / `transform: translateY()`
فقط — بدون Bounce، بدون دوران، بدون Flash، تمامًا كما هو مطلوب.

اكتمال كل المراحل يقع تحت **~850ms**، أي ضمن سقف الأمان الحالي (1500ms) وقريب
من الحد الأدنى للظهور (450ms) بحيث تُقرأ كلحظة هوية مقصودة وليست وميضًا، حتى
في حالة الإقلاع شديد السرعة من بيانات محلية.

## 4. سبب اختيار الألوان والهوية القانونية

- **الخلفية**: تدرّج شعاعي فاتح جدًا بين أبيض نقي (`#FFFFFF`) وبيج فاتح
  (`#F8F5EF` / `#F1EAD9`) — بدون أي لون داكن، مطابق لباقي هوية النظام.
- **الشعار**: تصميم SVG بالكامل (لا صور خارجية/PNG) يتكوّن من:
  - درع قانوني بسيط (Shield Path) بتدرّج كحلي داكن `#20334C → #0D1B2A`.
  - ميزان عدالة مبسّط بخطوط ذهبية داخل الدرع.
  - حلقة ذهبية رفيعة (`stroke-width: 1.6`) حول الشعار بتدرّج
    `var(--gold-lt) → var(--gold-dk)`.
- **الألوان العامة**: ذهبي (`var(--gold)` وتدرجاته)، كحلي/أزرق قانوني
  (`#0D1B2A`، `var(--info)` لأيقونة ورابط واتساب)، ورمادي فاتح للنصوص الثانوية
  (`var(--muted)`) — نفس لوحة الألوان المستخدمة في باقي شاشات "الحسام"، بدون
  أي ألوان صارخة.

## 5. كيف تم ضمان عدم تأثير الشاشة على سرعة الإقلاع

- لم يُضَف أي كود JavaScript جديد ولا أي `fetch`/`Promise`/`Timeout` إضافي —
  التعديل بصري (HTML/CSS) بحت.
- شاشة البداية لا تزال جزءًا من الـ markup الأولي في `index.html`، فتظهر فور
  أول رسم للصفحة دون انتظار تحميل أي JS.
- منطق `firstrun.js` (السقف 1500ms + الحد الأدنى 450ms المرتبط بجاهزية
  البيانات المحلية) لم يتغيّر إطلاقًا؛ الحركات الجديدة مصممة لتكتمل بصريًا خلال
  هذا الإطار الزمني نفسه بدلًا من أن تفرض إطارًا جديدًا.
- لا علاقة لأي من الحركات بـ Google Sync — الشاشة تختفي بمجرد جاهزية البيانات
  المحلية بالضبط كما كانت.

## 6. التحقق من التوافق مع جميع الشاشات

- تم تحديث `css/responsive.css` لتصغير الشعار والعناوين على الشاشات الصغيرة
  (`max-width: 768px`) بما يتناسب مع الكلاسات الجديدة.
- تصميم الشعار مبني بـ SVG بنظام `viewBox` (`0 0 120 120`) فيبقى حادًا وواضحًا
  عند أي تكبير للشاشة (Retina/4K) دون أي فقدان جودة، بعكس الرمز النصي القديم.
- التخطيط باستخدام `flex` بمحاذاة مركزية، فيبقى متجاوبًا على
  Desktop / Laptop / Tablet / Mobile بنفس الجودة دون الحاجة لأي media query
  إضافي بخلاف تصغير الأحجام.
- تمت إضافة `@media (prefers-reduced-motion: reduce)` تُلغي كل حركات
  Scale/TranslateY وتُبقي فقط Fade بسيط بمدة 300ms، تطبيقًا لمتطلبات
  إمكانية الوصول (Accessibility).
