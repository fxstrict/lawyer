# HOTFIX UX-03A — "ابدأ محلياً الآن" بدلاً من "تخطي"

## 0. نتيجة الـ Audit قبل التنفيذ

تمت قراءة `js/modules/firstrun.js` كاملاً، ثم مراجعة كل نصوص الـ Wizard في
`index.html` (السطور 37–72) وقسم "ربط Google Sheets" في صفحة الإعدادات
(السطور 319–340).

**اكتشاف مهم في السلوك الحالي قبل التعديل:** دالة `wizardSkip()` كانت
تعليقها التوثيقي يقول صراحة إنها **Session-only** — أي أنها لا تحفظ أي
شيء في `localStorage`، وبالتالي فإن `checkFirstRunWizard()` (المعتمدة فقط
على وجود/غياب `apiUrl`) كانت **تُعيد إظهار الـ Wizard في كل مرة يُعاد
فيها فتح البرنامج**، طالما لم يُحفظ رابط Google. هذا يخالف مباشرة الطلب
رقم 5 في هذا الهوتفكس، وكان لا بد من إصلاحه لتحقيق الهدف المطلوب — تم توثيق
ذلك بوضوح كإصلاح ضمن هذا الهوتفكس (وليس كإعادة هيكلة إضافية).

لا يوجد أي مكان آخر في المشروع يعرض نص "تخطي" مرتبطًا بـ Google Sync.

---

## 1. التغييرات المنفذة

### أولاً وثانياً — نص الزر والوصف (`index.html`)
- الزر: `تخطي الآن` → **`ابدأ محلياً الآن`**.
- تمت إضافة سطر توضيحي مباشرة فوق أزرار الويزارد (وليس استبدال نص
  الخصوصية الموجود، لأنه معلومة مختلفة ومفيدة أيضًا):
  > يمكنك استخدام البرنامج بالكامل على جهازك بدون أي اتصال خارجي.
  > عند رغبتك في النسخ الاحتياطي أو المزامنة بين الأجهزة يمكنك إضافة
  > رابط Google Apps Script لاحقاً من الإعدادات.

### ثالثاً — سلوك الزر (`js/modules/firstrun.js`)
أُعيدت تسمية `wizardSkip()` إلى **`wizardStartLocal()`** (المرجع الوحيد
لها في `index.html` تم تحديثه بنفس الاسم). سلوكها بالضبط كما طُلب:
- تُغلق الـ Wizard فورًا.
- **لا** تستدعي أي `fetch`.
- **لا** تحفظ أي رابط فارغ (لا تلمس `apiUrl` إطلاقًا).
- **لا** يظهر أي Loading Overlay (لم تكن موجودة أصلًا، وتم التحقق آليًا).
- الإضافة الوحيدة: `localStorage.setItem('localModeChosen','1')`.

### رابعاً — تنبيه صفحة الإعدادات (`index.html` + `js/modules/settings.js`)
عنصر `#localModeNotice` جديد داخل بطاقة "ربط Google Sheets"، بالنص وزر
"⚙️ إعداد المزامنة" المطلوبين بالضبط. إظهاره/إخفاؤه **لا يضيف دالة أو
مستمعًا جديدًا** — تم دمجه داخل `updateConnectionStatus()` الموجودة
أصلًا (وهي بالفعل الدالة الوحيدة التي تُستدعى عند كل تغيّر في حالة
الاتصال: عند الإقلاع، عند `saveApiUrl()`، وعند `wizardStartLocal()`).

### خامساً — عدم إعادة ظهور الـ Wizard (`js/modules/firstrun.js`)
```js
function checkFirstRunWizard() {
  ...
  if (!API_URL && !localStorage.getItem('localModeChosen')) { ... }
}
```
بمجرد ضغط "ابدأ محلياً الآن" مرة واحدة، لن تظهر الـ Wizard تلقائيًا مرة
أخرى في أي إقلاع لاحق، بغض النظر عن حالة `apiUrl`.

## 2. الملفات التي تم تعديلها

| الملف | التعديل |
|---|---|
| `js/modules/firstrun.js` | إعادة تسمية `wizardSkip→wizardStartLocal` + حفظ `localModeChosen` + تحديث شرط `checkFirstRunWizard`. |
| `index.html` | نص الزر، سطر توضيحي جديد في الويزارد، عنصر `#localModeNotice` في صفحة الإعدادات. |
| `js/modules/settings.js` | سطرين فقط داخل `updateConnectionStatus()` الموجودة أصلًا لإظهار/إخفاء `#localModeNotice`. |
| `js/tests/verify_firstrun_local_mode.js` | ملف اختبار جديد (Playwright، بنفس نمط `verify_runtime_wiring.js` الموجود في المشروع). |

لم يُلمس أي ملف آخر — لا Repository، لا DatabaseService، لا Undo، لا
Cache، لا Boot Sequence، لا `Code_v4.gs`.

## 3. التحقق — تشغيل فعلي على متصفح حقيقي (Chromium/Playwright)

`node js/tests/verify_firstrun_local_mode.js` → **12/12 ناجحة**، وتغطي
السيناريوهات السبعة المطلوبة بالضبط:

1. أول تشغيل بدون رابط Google → الويزارد مفتوح، نص الزر صحيح. ✅
2. الضغط على "ابدأ محلياً الآن" → الويزارد يُغلق فورًا، صفر Loading
   Overlay، **صفر طلبات شبكة لـ Google Apps Script** (تم رصد الشبكة
   فعليًا أثناء الاختبار، وليس افتراضًا). ✅
3. إعادة فتح البرنامج (Reload) → الويزارد **لا يظهر مجددًا**. ✅
4. إضافة الرابط لاحقًا من الإعدادات → تنبيه الوضع المحلي يختفي فورًا. ✅
5. (اختبار الاتصال الفعلي يتطلب خادم Apps Script حقيقي خارج نطاق بيئة
   الاختبار المعزولة — لكن التحقق من أن `saveApiUrl()` نفسها لا تُعطّل
   شيئًا ولا تُغيّر أي منطق قديم تم تأكيده، لأنها لم تُعدَّل إطلاقًا). ✅ *(انظر ملاحظة أدناه)*
6. حذف الرابط مرة أخرى (تفريغ الحقل وحفظه) → تنبيه الوضع المحلي يعود
   للظهور، بلا أي عطل. ✅
7. استمرار عمل البرنامج محليًا طوال الوقت → تم التنقل الفعلي بين صفحة
   الإعدادات ولوحة التحكم بعد كل خطوة أعلاه بنجاح. ✅

*ملاحظة على السيناريو 5:* "نجاح الاتصال" الفعلي يتطلب Web App منشور
وحقيقي على Google، وهو خارج نطاق أي بيئة اختبار آلي معزولة (كما هو الحال
في كل اختبارات المشروع السابقة لنفس السبب). ما تم التحقق منه آليًا هو أن
مسار الحفظ (`saveApiUrl`) والاتصال (`testConnection`) لم يتغيرا بحرف
واحد في هذا الهوتفكس، وأن واجهة المستخدم (التنبيه، الأزرار) تستجيب بشكل
صحيح لتغيّر `API_URL`.

## النتيجة

من أول ثانية: الزر يقول "ابدأ محلياً الآن" لا "تخطي"، والوصف يوضح أن
الربط اختياري، والاختيار يُحفظ فعليًا بحيث لا يُطارَد المستخدم بنفس
الشاشة في كل مرة، وصفحة الإعدادات تذكّره بلطف أن بإمكانه تفعيل المزامنة
متى أراد — دون أي تأثير على البيانات المحلية أو الأداء أو تسلسل الإقلاع.
